C言語の紹介

C言語は1972年にAT&Tベル研究所のデニス・リッチーによって開発された汎用プログラミング言語です。システムプログラミングやアプリケーション開発に広く使用されており、多くの現代プログラミング言語の基礎となっています。

C言語の特徴

  • 低レベルのメモリ操作が可能
  • シンプルなキーワードセット
  • 高速な実行速度
  • ポータビリティ（多くのプラットフォームで動作可能）
  • システムプログラミングに最適

最初のCプログラム

伝統的に、新しいプログラミング言語を学ぶ際の最初のプログラムは「Hello, World!」を表示するものです。以下がC言語での例です：

#include <stdio.h>

int main() {
    printf("Hello, World!\n");
    return 0;
}

このシンプルなプログラムは以下の要素で構成されています：

  • #include <stdio.h> - 標準入出力ライブラリをインクルード
  • int main() - プログラムの実行が開始される主要関数
  • printf() - 文字列を出力する関数
  • return 0; - プログラムが正常に終了したことを示す

変数とデータ型

C言語では、データを保存するために変数を使用します。各変数には特定のデータ型があり、どのような種類の値を保存できるかを定義します。

基本データ型

データ型 説明 サイズ (一般的)
int 整数 4バイト
float 単精度浮動小数点 4バイト
double 倍精度浮動小数点 8バイト
char 文字 1バイト
void 値なし -

変数の宣言と初期化

C言語では、変数を使用する前に宣言する必要があります。変数の宣言と初期化の例：

int age = 25; // 整数型
float price = 19.99; // 単精度浮動小数点型
double pi = 3.14159265359; // 倍精度浮動小数点型
char grade = 'A'; // 文字型
char name[20] = "田中太郎"; // 文字列（文字の配列）

型修飾子

C言語では、基本データ型を修飾して、サイズや符号を変更できます：

  • short - サイズを小さくする（例：short int）
  • long - サイズを大きくする（例：long int）
  • signed - 符号付き（デフォルト）
  • unsigned - 符号なし（正の値のみ）
unsigned int positive_num = 50000;
long long int very_big_num = 9223372036854775807LL;
unsigned char byte = 255;

制御構造

制御構造は、プログラムの実行フローを制御するためのステートメントです。C言語には、条件分岐とループのための様々な制御構造があります。

条件分岐

if-else文

int age = 20;

if (age >= 20) {
    printf("成人です。\n");
} else if (age >= 18) {
    printf("18歳以上です。\n");
} else {
    printf("未成年です。\n");
}

switch文

char grade = 'B';

switch (grade) {
    case 'A':
        printf("優秀です！\n");
        break;
    case 'B':
        printf("良好です。\n");
        break;
    case 'C':
        printf("平均的です。\n");
        break;
    default:
        printf("評価不明です。\n");
}

ループ

for ループ

// 1から10までの数字を表示
for (int i = 1; i <= 10; i++) {
    printf("%d ", i);
}
printf("\n");

while ループ

int count = 1;

while (count <= 5) {
    printf("カウント: %d\n", count);
    count++;
}

do-while ループ

int num;

do {
    printf("1から10の間の数字を入力してください: ");
    scanf("%d", &num);
} while (num < 1 || num > 10);

制御ステートメント

  • break - ループやswitch文から抜ける
  • continue - ループの現在の反復をスキップして次の反復に進む
  • goto - 指定されたラベルに制御を移す（一般的には避けるべき）

関数

関数は、特定のタスクを実行するためにまとめられたコードのブロックです。関数を使用することで、コードの再利用性と可読性が向上します。

関数の定義と呼び出し

// 関数の定義
int add(int a, int b) {
    return a + b;
}

int main() {
    // 関数の呼び出し
    int result = add(5, 3);
    printf("5 + 3 = %d\n", result);
    return 0;
}

関数プロトタイプ

関数プロトタイプは、関数の定義の前に関数の宣言を行うことで、コンパイラに関数の存在を知らせます。

// 関数プロトタイプ
int multiply(int x, int y);

int main() {
    int result = multiply(4, 5);
    printf("4 * 5 = %d\n", result);
    return 0;
}

// 関数の定義
int multiply(int x, int y) {
    return x * y;
}

引数の渡し方

C言語では、引数は「値渡し」がデフォルトです。関数内で引数の値を変更しても、呼び出し元の変数には影響しません。

値渡し

void increment(int num) {
    num++; // この変更は呼び出し元には影響しない
}

int main() {
    int x = 10;
    increment(x);
    printf("x = %d\n", x); // 出力: x = 10
    return 0;
}

参照渡し（ポインタを使用）

void increment_by_reference(int *num) {
    (*num)++; // この変更は呼び出し元に影響する
}

int main() {
    int x = 10;
    increment_by_reference(&x);
    printf("x = %d\n", x); // 出力: x = 11
    return 0;
}

配列とポインタ

配列

配列は、同じデータ型の要素のコレクションです。C言語では、配列のインデックスは0から始まります。

// 整数の配列を宣言して初期化
int numbers[5] = {10, 20, 30, 40, 50};

// 配列の要素にアクセス
printf("3番目の要素: %d\n", numbers[2]); // 出力: 30

// 配列の要素を変更
numbers[1] = 25;
printf("2番目の要素: %d\n", numbers[1]); // 出力: 25

// 多次元配列
int matrix[3][3] = {
    {1, 2, 3},
    {4, 5, 6},
    {7, 8, 9}
};

printf("matrix[1][2]: %d\n", matrix[1][2]); // 出力: 6

ポインタ

ポインタは、メモリアドレスを格納する変数です。C言語でのポインタの使用は強力ですが、注意が必要です。

int num = 42;
int *ptr; // ポインタ変数の宣言

ptr = # // numのアドレスをptrに格納

printf("num の値: %d\n", num); // 出力: 42
printf("num のアドレス: %p\n", &num); // 例: 0x7ffeeb7a35bc
printf("ptr の値: %p\n", ptr); // 例: 0x7ffeeb7a35bc
printf("ptr が指す値: %d\n", *ptr); // 出力: 42

// ポインタを使って変数の値を変更
*ptr = 100;
printf("変更後の num の値: %d\n", num); // 出力: 100

配列とポインタの関係

C言語では、配列名は配列の最初の要素へのポインタとして機能します。

int arr[5] = {10, 20, 30, 40, 50};
int *p = arr; // arrは&arr[0]と同じ

printf("arr[0]: %d\n", arr[0]); // 出力: 10
printf("*p: %d\n", *p); // 出力: 10
printf("*(p+1): %d\n", *(p+1)); // 出力: 20
printf("p[2]: %d\n", p[2]); // 出力: 30

動的メモリ割り当て

C言語では、malloc()、calloc()、realloc()、free()関数を使用して、実行時にメモリを動的に割り当てることができます。

#include <stdlib.h>

// 動的に整数配列を作成
int *dynamicArray = (int *)malloc(5 * sizeof(int));

if (dynamicArray == NULL) {
    printf("メモリ割り当て失敗\n");
    return 1;
}

// 配列に値を設定
for (int i = 0; i < 5; i++) {
    dynamicArray[i] = i * 10;
}

// 配列の値を表示
for (int i = 0; i < 5; i++) {
    printf("%d ", dynamicArray[i]); // 出力: 0 10 20 30 40
}
printf("\n");

// メモリを解放
free(dynamicArray);

構造体

構造体は、異なるデータ型の変数をグループ化するためのユーザー定義のデータ型です。

構造体の定義と使用

// 構造体の定義
struct Student {
    char name[50];
    int id;
    float gpa;
};

int main() {
    // 構造体変数の宣言と初期化
    struct Student s1 = {"佐藤健太", 12345, 3.8};
    
    // 構造体のメンバーにアクセス
    printf("名前: %s\n", s1.name);
    printf("学生ID: %d\n", s1.id);
    printf("GPA: %.1f\n", s1.gpa);
    
    // 構造体のメンバーを変更
    s1.gpa = 4.0;
    printf("新しいGPA: %.1f\n", s1.gpa);
    
    return 0;
}

構造体配列

struct Student students[3] = {
    {"佐藤健太", 12345, 3.8},
    {"鈴木花子", 12346, 3.9},
    {"田中太郎", 12347, 3.7}
};

for (int i = 0; i < 3; i++) {
    printf("学生 %d: %s, ID: %d, GPA: %.1f\n",
        i+1, students[i].name, students[i].id, students[i].gpa);
}

構造体ポインタ

struct Student s1 = {"佐藤健太", 12345, 3.8};
struct Student *ptr = &s1;

// ポインタを使って構造体のメンバーにアクセス
printf("名前: %s\n", ptr->name); // アロー演算子を使用
printf("学生ID: %d\n", (*ptr).id); // ドット演算子を使用
printf("GPA: %.1f\n", ptr->gpa);

typedef を使用した構造体

typedefを使用すると、構造体に新しい名前を付けることができ、structキーワードを省略できます。

typedef struct {
    char title[100];
    char author[50];
    int year;
} Book;

Book myBook = {"C言語プログラミング", "山田一郎", 2023};
printf("タイトル: %s\n", myBook.title);
printf("著者: %s\n", myBook.author);
printf("出版年: %d\n", myBook.year);

ファイル入出力

C言語では、stdio.hライブラリの関数を使用して、ファイルの読み書きを行うことができます。

ファイルを開く・閉じる

#include <stdio.h>

int main() {
    FILE *file;
    
    // ファイルを書き込みモードで開く
    file = fopen("example.txt", "w");
    
    if (file == NULL) {
        printf("ファイルを開けませんでした\n");
        return 1;
    }
    
    // ファイルに書き込む処理...
    
    // ファイルを閉じる
    fclose(file);
    
    return 0;
}

ファイルへの書き込み

FILE *file = fopen("example.txt", "w");

if (file != NULL) {
    fprintf(file, "こんにちは、C言語!\n");
    fprintf(file, "ファイル入出力のテストです。\n");
    
    // 整数と浮動小数点数を書き込む
    int num = 42;
    float pi = 3.14159;
    fprintf(file, "数値: %d, %.2f\n", num, pi);
    
    fclose(file);
}

ファイルからの読み込み

FILE *file = fopen("example.txt", "r");

if (file != NULL) {
    char buffer[100];
    
    // 1行ずつ読み込む
    while (fgets(buffer, sizeof(buffer), file) != NULL) {
        printf("%s", buffer);
    }
    
    fclose(file);
}

バイナリファイルの操作

// 構造体を定義
typedef struct {
    int id;
    char name[50];
    float salary;
} Employee;

// バイナリファイルに書き込み
FILE *file = fopen("employees.bin", "wb");

if (file != NULL) {
    Employee emp1 = {1, "佐藤健太", 50000.0};
    Employee emp2 = {2, "鈴木花子", 55000.0};
    
    fwrite(&emp1, sizeof(Employee), 1, file);
    fwrite(&emp2, sizeof(Employee), 1, file);
    
    fclose(file);
}

// バイナリファイルから読み込み
file = fopen("employees.bin", "rb");

if (file != NULL) {
    Employee emp;
    
    while (fread(&emp, sizeof(Employee), 1, file) == 1) {
        printf("ID: %d, 名前: %s, 給与: %.2f\n", emp.id, emp.name, emp.salary);
    }
    
    fclose(file);
}

メモリ管理

C言語では、メモリ管理が非常に重要です。適切なメモリ管理を行わないと、メモリリークやセグメンテーション違反などの問題が発生する可能性があります。

メモリレイアウト

C言語のプログラムは、一般的に以下のメモリセグメントに分かれています：

  • テキストセグメント：プログラムのコード
  • 初期化済みデータセグメント：初期化された静的変数とグローバル変数
  • 未初期化データセグメント（BSS）：初期化されていない静的変数とグローバル変数
  • ヒープ：動的メモリ割り当て（malloc、calloc、realloc）
  • スタック：関数呼び出し、ローカル変数

動的メモリ割り当て関数

主なメモリ割り当て関数

  • malloc(size_t size)：指定されたサイズのメモリブロックを割り当て
  • calloc(size_t num, size_t size)：指定された数と大きさの要素のためのメモリを割り当て、0で初期化
  • realloc(void *ptr, size_t size)：既存のメモリブロックのサイズを変更
  • free(void *ptr)：動的に割り当てられたメモリを解放

メモリリークの防止

メモリリークを防ぐためには、動的に割り当てられたメモリを使用し終わったら必ず解放することが重要です。

#include <stdlib.h>

int main() {
    // メモリを割り当て
    int *arr = (int *)malloc(10 * sizeof(int));
    
    if (arr == NULL) {
        return 1; // メモリ割り当て失敗
    }
    
    // メモリを使用
    for (int i = 0; i < 10; i++) {
        arr[i] = i * 10;
    }
    
    // 処理が終わったらメモリを解放
    free(arr);
    arr = NULL; // 解放後はNULLを設定（ダングリングポインタの防止）
    
    return 0;
}

一般的なメモリ関連の問題

  • メモリリーク：割り当てられたメモリが解放されない
  • ダングリングポインタ：解放されたメモリを指し続けるポインタ
  • バッファオーバーフロー：割り当てられたメモリの境界を超えてアクセス
  • 二重解放：同じメモリブロックを複数回解放
  • メモリ断片化：空きメモリが小さな断片に分かれてしまう

注意点

C言語ではメモリ管理は開発者の責任です。適切なメモリ管理を行わないと、プログラムのクラッシュやセキュリティの脆弱性につながる可能性があります。
